# 도구 이해하기

Hermes가 실제로 일을 할 수 있는 이유는 **도구(tool)**를 사용할 수 있기 때문입니다.

## 자주 쓰는 도구

| 도구 | 쉬운 설명 |
| --- | --- |
| 파일 | 문서와 코드 읽기·수정 |
| 터미널 | 명령 실행, 프로그램 설치, 테스트 |
| 웹 | 최신 정보 검색과 웹페이지 내용 확인 |
| 브라우저 | 웹사이트를 직접 열고 클릭·입력 |
| 이미지 | 이미지 생성 또는 분석 |
| 일정 | 정해진 시간에 반복 작업 실행 |

## 도구 세트란?

도구가 너무 많으면 복잡하거나 불필요한 권한이 생길 수 있습니다. 그래서 비슷한 도구를 묶은 **도구 세트(toolset)**를 켜고 끌 수 있습니다.

터미널에서 도구를 관리하려면:

```bash
hermes tools
```

## 안전하게 요청하는 방법

실제로 바뀌는 작업은 범위를 분명히 말하는 편이 좋습니다.

- “이 파일을 읽고 요약만 해줘”
- “테스트는 실행하되, 파일은 수정하지 마”
- “변경 전에 어떤 파일을 수정할지 먼저 알려줘”

Hermes는 작업 결과를 실제 도구 출력으로 확인한 뒤 알려주는 것이 원칙입니다.
